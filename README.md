# 🏖 Планировщик отпуска

Одностраничное приложение для планирования отпуска. Позволяет собирать информацию о локациях (отели + координаты), добавлять интересные места рядом (рестораны, прогулки, достопримечательности), строить маршруты по дням и открывать их в Яндекс.Картах.

Работает полностью на клиенте — данные хранятся в `localStorage` браузера. Публикуется на GitHub Pages.

## Возможности

- **Локации** — добавление отелей/проживания с координатами
- **Категории мест:**
  - 🍽 Где поесть
  - 🚶 Где погулять
  - 👁 Что посмотреть
- **Поиск мест через Яндекс.Карты** — ввод названия → подсказки с геокодера → добавление одним кликом
- **Интерактивная карта** — отель и все места отображаются маркерами
- **Планировщик маршрутов по дням** — перетаскивание мест, заметки на день
- **Построение маршрута** — формирование ссылки Яндекс.Карт с точками в порядке маршрута
- **Drag & drop** — сортировка мест внутри категорий и в маршрутах
- **Импорт/экспорт данных** (JSON) — резервное копирование и перенос между устройствами
- **Хранение** — стартовые данные из `data/example.json`, изменения сохраняются в `localStorage`

## Технологии

- **Vanilla JS** (ES-модули) — без фреймворков и сборщиков
- **Yandex Maps JavaScript API 3.0** — карта, маркеры, геокодинг
- **HTML5 Drag & Drop API** — сортировка перетаскиванием
- **localStorage** — хранение данных
- **CSS Custom Properties** — темизация и цвета категорий

## Как запустить

Приложение использует ES-модули (`import`/`export`), поэтому не работает при открытии `index.html` через `file://`. Требуется локальный сервер:

```bash
# Вариант 1: npx serve
npx serve .

# Вариант 2: Python
python -m http.server 8000

# Вариант 3: Node.js
npx http-server .
```

Откройте `http://localhost:3000` (или другой порт) в браузере.

## Получение API-ключа Яндекс.Карт

1. Перейдите в [кабинет разработчика Яндекс.Технологии](https://developer.tech.yandex.ru/services/)
2. Подключите **JavaScript API 3.0**
3. Создайте ключ с ограничением по HTTP Referrer:
   - `localhost:*` — для локальной разработки
   - `*.github.io` — для GitHub Pages
   - `ваш-домен.ru/*` — если используете собственный домен
4. Впишите ключ в `config.js`:

```js
export const APP_CONFIG = {
  ymapsApiKey: 'ВАШ_API_КЛЮЧ'
};
```

### Защита API-ключа

Ключ будет виден в инструментах разработчика браузера — это норма для статического сайта. Основной способ защиты — **ограничение по HTTP Referrer** в кабинете разработчика. Без него ключ сможет использовать кто угодно с любого сайта.

## Публикация на GitHub Pages

1. Создайте репозиторий на GitHub
2. Залейте файлы проекта в корень (ветка `main`)
3. Перейдите: Settings → Pages → Source: **Deploy from a branch**
4. Ветка: `main`, папка: `/ (root)`
5. Через 1–2 минуты сайт будет доступен по адресу `https://ваш-username.github.io/имя-репозитория/`
